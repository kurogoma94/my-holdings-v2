# Amazon検索キーワード（バックエンド）最適化提案

**対象商品：エリソース・シークヮーサー**

Amazon公式の最新ガイドライン（2026年時点）に基づき、検索順位を最大化するためのバックエンドキーワード（検索キーワード欄）を作成しました。

## 1. 設定用キーワード（2行目〜5行目）

1行目は設定済みとのことですので、残りの4行分（各行約120バイト・40文字程度）を作成しました。各行の内容をコピー＆ペーストして設定してください。

### 2行目：利用シーン・ターゲット料理

```text
サーモン 寿司 刺身 冷しゃぶ 唐揚げ エビフライ フィッシュアンドチップス サラダ ドレッシング
```

### 3行目：味の表現・代替品ニーズ

```text
ステーキ 焼き肉 ハンバーグ 揚げ物 タルタルソース 代用 マヨネーズ ベース 柑橘 さっぱり
```

### 4行目：ベネフィット・ブランド価値

```text
フルーツ 隠し味 コク 旨味 酸味 スパイシー ハーブ にんにく お土産 プレゼント ギフト
```

### 5行目：利便性・表記揺れ対策

```text
お取り寄せ キャンプ 贈り物 パーティー 自宅 贅沢 専門店 プロの味 シークワーサー しいくわあさあ シークワサー 柑橘ソース 常温保存 沖縄土産
```

## 2. SEO対策のポイント

- **分散入力のメリット**: Amazonのシステム上、100バイト×5行でも、1つのフィールドに500バイトでも、インデックス（検索対象化）の扱いは同じですが、視認性と管理のしやすさから分けて入力するのが推奨されます。
- **500バイト制限（全行合計）**: 5つの行の**合計バイト数**が500バイト（全角約166文字）以内である必要があります。今回は1行目を含めても450バイト程度に収まるよう設計しています。
- **重複の排除**: 1行目に既に入力されている単語や、商品タイトルにある単語は、この2〜5行目からは徹底して排除しています。
- **お悩み・利用シーンへの訴求**: 商品名ではカバーしきれない「サーモン」「冷しゃぶ」「お土産」などの具体的な食べ方や検索意図を盛り込んでいます。
- **表記揺れ対策**: 「シークワーサー」「シークヮーサー（タイトル込）」「しいくわあさあ」など、ユーザーが検索しそうなバリエーションを網羅しました。

## 3. 注意事項

- **句読点は不要**: カンマや句点を使わず、半角スペースで区切っています（Amazon推奨形式）。
- **ブランド名は除外**: 自社・他社ブランド名を入れるのは規約違反となるため、含めていません。
